# 피드백 작성 팁

## 우선순위 가이드:
- **HIGH**: 즉시 처리, 다른 목표보다 우선
  - 예: 치명적 버그, 비용 폭주, 보안 문제
- **MEDIUM**: 현재 목표에 통합
  - 예: 효율 개선, 기능 추가
- **LOW**: 백로그 추가
  - 예: 장기 개선, nice-to-have

## 좋은 피드백 예시:
✅ "토큰 사용량 30% 줄여줘 - 구체적으로 state.yaml 전체 읽기 대신 quick_state.py 사용"
✅ "배치 실행 모드 추가 - 독립적 작업 3개까지 한 사이클에 처리"
✅ "novelty_checker.py 임계값 0.3 → 0.5로 조정하고 재테스트"

## 피해야 할 피드백:
❌ "더 잘해" - 너무 모호
❌ "버그 고쳐" - 어떤 버그인지 불명확
❌ "최적화 해줘" - 무엇을 어떻게?

## 효과적인 피드백 구조:
1. **구체적**: "X를 Y로 바꿔"
2. **측정 가능**: "토큰 30% 감소"
3. **실행 가능**: 명확한 행동 항목
4. **근거 있음**: 왜 필요한지 설명
5. **테스트 가능**: 성공 여부 판단 기준
