# Human Feedback 016: 검증 정직성 - 거짓 완료 선언 금지

**From:** 김경태
**Date:** 2025-12-08
**Priority:** CRITICAL

## 문제

new.giz.ai E2E 검증 "42/42 통과" 선언했으나 **실제 로그인 불가**.
이건 심각한 신뢰 문제다.

## 근본 원인

1. **Playwright 테스트 = 실제 검증 아님** - 테스트 코드가 통과해도 실서비스 작동 보장 안됨
2. **자기 평가의 한계** - 자신이 만든 테스트로 자신을 검증하면 맹점 발생
3. **완료 압박** - 빨리 끝냈다고 보고하고 싶은 유혹

## 재발 방지 규칙 (GENOME에 반영할 것)

### 1. 실서비스 수동 검증 필수
- E2E 테스트 통과 후 **반드시 chrome-devtools로 실제 사이트 접속**
- 최소 체크: 로그인, 핵심 기능 1개 직접 클릭

### 2. 검증 증거 첨부
- 스크린샷 또는 console 로그 필수
- "테스트 통과"만으로 완료 선언 금지

### 3. 불확실하면 "미확인" 표기
- 확신 없으면 솔직히 "테스트는 통과했으나 실서비스 미확인"
- 과장/거짓 완료보다 정직한 미완료가 100배 낫다

### 4. 자기 평가 상한 강화
- 외부 검증(인간 확인, 실서비스 테스트) 없이 자기 평가 0.3 상한
- 실서비스 수동 검증 완료 시 0.5까지
- 인간 최종 확인 후에만 1.0

## 즉시 조치

1. state.yaml의 e2e_verified 상태 취소
2. new.giz.ai 실제 접속해서 로그인 문제 원인 파악
3. 문제 해결 후 **chrome-devtools로 직접 검증**하고 증거와 함께 보고

신뢰는 한번 깨지면 복구 어렵다. 정직하게.
